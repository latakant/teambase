# /cortex-generate — Skill 47
# LOAD WHEN: /cortex-generate (any mode)
# Code generation from spec. Business logic is always preserved.
# Providers: Razorpay · Resend · Cloudinary (detected from package.json)

$ARGUMENTS

## Usage
cortex generate full          → generate from all spec files
cortex generate delta         → generate only what spec has but code lacks
cortex generate preview       → show what would be generated (no writes)
cortex generate check         → verify code matches spec (score only)
cortex generate <entity>      → generate one entity only
cortex generate --frontend    → generate Next.js page from contract spec

## TOKEN BUDGET (this skill)
Load only: spec for target entity + CRITICAL policies + matching generator.
Do NOT load all spec files for a single entity generation.

---

## Generation Protocol

Step 1: Load spec/policies/CRITICAL.yaml (Tier 1 — always OK)
Step 2: Load spec for target only:
  entity:   ai/spec/domain/entities/<entity>.yaml
  contract: ai/spec/contracts/api/<domain>-api.yaml (if exists)
Step 3: Load generator:
  backend:  ai/generators/nestjs/generator.md
  frontend: ai/generators/nextjs/generator.md
  schema:   ai/generators/prisma/generator.md

Step 4: Check if target file already exists
  If YES + has business logic → ⛔ HARD HALT (POL-CRIT-007)
  If YES + is pure skeleton   → confirm with user before overwrite
  If NO                       → generate

Step 5: Generate with header:
  // GENERATED BY CORTEX v7.1 LEAN SOVEREIGN
  // Spec: ai/spec/domain/entities/<entity>.yaml
  // DO NOT edit structure — regenerate via: /cortex-generate <entity>
  // Business logic: implement methods marked [IMPLEMENT]

Step 6: Run npx tsc --noEmit — fix any type errors before reporting
Step 7: Report what was generated + next steps

---

## Exena-Specific Generation Rules

Payment domain:
  Always use Razorpay (not Stripe)
  Always $transaction for Payment + Order updates
  Always HMAC-SHA256 webhook verification

Email domain:
  Always Resend via BullMQ queue
  Never import MailerModule — inject MailerService directly

Storage domain:
  Always Cloudinary
  Always store publicId alongside URL

Auth domain:
  JWT_SECRET + JWT_REFRESH_SECRET (separate values)
  OTP flow via MSG91 for customers
  email/password bcrypt for admin

---

## HARD HALT conditions
⛔ POL-CRIT-007: File has business logic — will not overwrite
   Action: Delete skeleton file first, then regenerate
⛔ POL-CRIT-001: Spec not PA-approved — will not generate
   Action: Run /cortex-spec propose, get PA approval first
