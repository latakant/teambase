# HUMAN MODE — Shared Protocol
# v1.0 · Cortex v19.0
# Add `--human` to any cert-* skill to translate output to plain English.
# Referenced by: cert-verify · cert-score · cert-bug · cert-feature · all cert-*

---

## What --human does

Runs the skill normally. Then translates all findings into plain English decisions.
No technical jargon. No TypeScript errors. Just: what is wrong, what is the risk, what to do.

Writes translated output to both:
- Terminal (immediate read)
- `ai/reports/human-summary.md` (persisted — share with non-dev stakeholders)

## Translation rules (dev output → human output)

| Dev finding | Human translation |
|---|---|
| `TS-007: P2025 not mapped to 404` | "If a user looks up something that doesn't exist, the app crashes instead of showing 'not found'" |
| `tsc --noEmit: 3 errors` | "The code has 3 technical errors that will prevent the app from compiling" |
| `SEC-03: rate limit missing` | "Users could spam the login page — no protection against repeated attempts" |
| `score: 87/100 WATCH` | "The project is in caution zone — only bug fixes allowed until score improves" |
| `score: 72/100 BLOCK` | "The project has significant quality issues — no new features until fixed" |
| `BL-01 violated` | "[Plain English from business-rules.md `human:` field for BL-01]" |
| `jest: 2 failed` | "2 automated tests are failing — these are checks that verify the app behaves correctly" |
| `secret leaked: JWT_SECRET` | "A security key was accidentally included in the code — must be removed immediately" |

When translating business rule violations: read the `human:` field from `ai/context/business-rules.md` for the rule ID. Use that exact text.

---

## Human-readable severity scale

| Technical | Human |
|---|---|
| BLOCK 🚫 | "Must fix before launch" |
| WARN ⚠️ | "Should fix this sprint" |
| INFO ℹ️ | "Worth knowing, not urgent" |
| PASS ✅ | "Working correctly" |

## Writing to ai/reports/human-summary.md

```markdown
# Health Report — [project] · [date]
Generated by: /[skill-name] --human · Cortex v19.0

## Status: [GREEN / YELLOW / RED]
Score: [X]/100

## What's working
[list]

## What needs attention
[list with When tags]

## What's broken
[list]

## Business rules
[N verified / N at risk / N violated]

## Recommended action
[one sentence]
```

Append to the file if it exists. Overwrite section for same date.

---

## How to add Human Mode to any cert-* skill

Add this block at the end of the skill, before the completion block:

```markdown
---

## HUMAN MODE (fires if `--human` in arguments)

After all steps complete, translate findings to plain English using the
protocol in `skills/common/human-mode.md`.

Write to terminal + `ai/reports/human-summary.md`.
```
