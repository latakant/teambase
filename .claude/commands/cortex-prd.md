# cert-prd — PRD Generator / Updater
> Generates or refreshes app.prd.md for ANY project from the codebase.
> Reads code → infers intent → writes structured product spec.
> Run after /cortex-discover or standalone.

---

## When to Use

| Situation | Command |
|-----------|---------|
| New project, no PRD yet | `/cert-prd generate` |
| PRD exists but is stale | `/cert-prd refresh` |
| Feature was just added | `/cert-prd update [module]` |
| Want to see what's built vs specced | `/cert-prd diff` |

---

## MODE: GENERATE (no existing PRD)

### Step 1 — Load intelligence
If `ai/PROJECT_INTELLIGENCE.md` exists → read it (Phase 3-5 data).
If not → run Phases 1-5 of `/cortex-discover` inline (no need for full discover).

### Step 2 — Build feature registry

For each module/controller found:
```
Module: [name]
  Endpoints: [list with method + path + auth level]
  Business purpose: [inferred from code]
  Status: BUILT | PARTIAL | STUB
  Test coverage: EXISTS | MISSING
```

### Step 3 — Infer business rules

Read service files. For each non-trivial function:
- What precondition does it check? → business rule
- What does it calculate? → business logic
- What does it queue or email? → side effect rule

### Step 4 — Identify scope gaps

Compare feature registry against business type checklist (from /cortex-discover Phase 6).
Every gap is a potential future feature to spec.

### Step 5 — Write ai/app.prd.md

```markdown
# [Project Name] — Product Requirements Document
Version: 1.0 | Generated: [date] by /cert-prd | Project type: [type]

## Overview
[2-3 sentences: what is this, who is it for, current status]

## Tech Stack
[from PROJECT_INTELLIGENCE.md]

## Feature Registry — [N] modules

### [Module Name]
**Purpose:** [one line]
**Status:** BUILT / PARTIAL / STUB
**Endpoints:**
  - [METHOD] /api/[path] — [purpose] — [auth level]
**Business rules:**
  - [rule 1]
  - [rule 2]
**Test coverage:** [Y/N]

[repeat for each module]

## Business Rules (cross-module)
[rules that span multiple modules]

## External Integrations
[each integration: purpose + webhook if any]

## Current Gaps
[from Phase 6 gap analysis]

## Decision Log
[key technical decisions and why — inferred from code patterns]

## Changelog
[date] GENERATED by /cert-prd from codebase scan
```

## MODE: REFRESH (existing PRD)

Read current `ai/app.prd.md`.
Re-scan codebase for changes since last generation date.
For each change found:
- New endpoint → add to feature registry
- Deleted endpoint → mark as REMOVED
- New model → add to relevant module section
- New business rule inferred → add to rules section

Output diff:
```
PRD REFRESH DIFF
  Added:   [list]
  Removed: [list]
  Changed: [list]
  Unchanged: [N sections]
```

Append to changelog section:
```
[date] REFRESHED by /cert-prd — added: [N], removed: [N], changed: [N]
```

---

## MODE: UPDATE [module] (targeted update)

Only re-scan the specified module's files.
Update only that module's section in app.prd.md.
Faster than full refresh for post-feature-build updates.

---

## MODE: DIFF (spec vs reality check)

Read app.prd.md. Scan codebase. Output:

```
PRD vs CODEBASE DIFF
═══════════════════════════════════════════
  In PRD but NOT in code:   [list — planned but unbuilt]
  In code but NOT in PRD:   [list — built but undocumented]
  Matching:                 [N features]
═══════════════════════════════════════════
Recommendation: run /cert-prd refresh to sync
```
